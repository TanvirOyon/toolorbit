Compressing a PDF sounds simple until you do it and the file comes back nearly the same size - or noticeably worse quality. The problem is that "compress PDF" means different things depending on what is inside the PDF. This guide explains exactly what is happening and what method to use in each situation.

## What makes a PDF large?

A PDF file is a container for several different types of content, each of which contributes to file size differently.

**Embedded images** are usually the biggest contributor. A PDF generated from a Word document with a few high-resolution photos can easily be 10MB or more, with the images accounting for 90% of that size. The images may be stored as JPEG, PNG, or raw bitmap data inside the PDF object tree.

**Embedded fonts** are the second contributor. A PDF that embeds a full font file - rather than just the characters actually used - can add 200–500KB per typeface. Modern PDF generators usually subset fonts (include only the glyphs present in the document), but older generators and some design tools embed complete font files.

**PDF structure overhead** - the cross-reference table, object headers, and metadata - is usually small but can be bloated in PDFs generated by old software that used uncompressed cross-reference tables.

## Method 1: Structural optimization (fastest, safest)

This is what ToolOrbit's [PDF Optimizer](/tools/pdf-compress) does. It rewrites the PDF's cross-reference table as a compressed object stream. This is a completely lossless operation - no content is changed, no images are re-encoded, no fonts are modified.

**Best for:** PDFs generated by older software (pre-2015 versions of InDesign, older print drivers, legacy government e-forms systems).

**Typical result:** 5–30% size reduction. Can be zero or slightly negative (the file gets marginally larger) if the PDF was already optimized.

**Privacy advantage:** Runs entirely in your browser. Your document never leaves your device.

## Method 2: Image re-encoding (biggest size reduction, some quality trade-off)

This re-encodes embedded JPEG images at a lower quality setting. A photo saved at 95% JPEG quality in your PDF might be re-encoded at 80%, reducing that image's contribution to file size by 50–70%.

**Best for:** Presentation PDFs, portfolios, product catalogs - any PDF where the bulk of the size comes from photographic images.

**Typical result:** 40–80% total size reduction for photo-heavy PDFs.

**Trade-off:** The re-encoding introduces additional JPEG compression artifacts. At 80% quality this is usually imperceptible; below 60% it becomes visible, particularly on text overlaid on image backgrounds.

**How to do it:**
- Ghostscript (command line): `gs -sDEVICE=pdfwrite -dCompatibilityLevel=1.4 -dPDFSETTINGS=/ebook -dNOPAUSE -dQUIET -dBATCH -sOutputFile=output.pdf input.pdf`
- Adobe Acrobat Pro: File → Reduce File Size or Save As Optimized PDF
- iLovePDF, Smallpdf (server-side - your file is uploaded)

## Method 3: Font subsetting

If the PDF embeds full font files rather than subsets, removing the unused glyphs can save significant space. This is rare in modern PDFs but common in PDFs exported from older desktop publishing software.

**Best for:** PDFs with multiple embedded fonts and relatively few unique characters.

**How to detect:** Open the PDF in Acrobat (or a free alternative like PDF-XChange Viewer), go to File Properties → Fonts. If you see font names without "(Embedded Subset)" - just "Embedded" - those are full font embeddings that could be subsetted.

## Which method should I use?

| Your PDF | Best method |
|---|---|
| Generated by modern Word/Google Docs | Structural optimization (method 1) - likely already optimized |
| Contains high-resolution photos | Image re-encoding (method 2) |
| Was made by software older than 2015 | Start with method 1, then method 2 if needed |
| Is a scanned document | Image re-encoding is the only meaningful option |
| Contains sensitive information | Use a browser-based tool (ToolOrbit) - no upload |

## The "compress PDF" button that does nothing

Many online tools claim to compress PDFs but apply only structural optimization - which, as explained above, has minimal effect on modern PDFs. If you upload a 5MB PDF with high-resolution photos and get back a 4.9MB PDF, this is what happened. The tool technically worked as described, but the description was misleading.

The only way to significantly reduce a large, image-heavy PDF is to re-encode the images. This requires server-side processing (Ghostscript or Adobe's engine) or a local desktop application. A browser-only tool cannot do it without quality loss because re-encoding at scale requires compute that is impractical client-side.

## Compress before you need to

The best time to think about PDF compression is when you create the document, not after. In Word's PDF export dialog, choose "Minimum size (publishing online)" rather than "Best quality" for documents intended for email or web upload. In Photoshop or Illustrator, flatten the file and reduce image resolution to 150 DPI before exporting to PDF. These choices at creation time produce smaller files than any post-hoc compression can achieve.
